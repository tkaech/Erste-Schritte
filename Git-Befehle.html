Nachfolgend ein paar Git-Befehle

git init //erstellt ein neues Repository git add . //fügt alle Dateien dem Repository hinzu git add Git-Befehle.html //fügt nur die genannte Datei hinzug git status // Zeigt den aktuellen Status an

Commit durchführen

git commit -m "add Git-Befehle" git log //Listet die Commits auf git log --pretty=oneline // Ansicht wird so kompakter angezeigt git log --oneline // Ansicht auf einer Linie ohne ganzen Hash

Diff-Befehl

git diff .\Git-Befehle.html //Unterschied zur angegebenen Datei. Zuvor speichern, es wird nur die gespeicherte Datei verglichen und nicht die im Memory geändert. git revert 75e9de0e3370d5a40d0f5e2e79d671bb43b3f78d // zu einem früheren Zeitpunkt zurückspringen git revert --continue //bei einem Konflikt wird beim nächsten Commit die Wahl gestellt, ob weiter oder abbruch git revert --abort git reset --hard HEAD~2 //löscht die letzten zwei Commits git reset --soft HEAD~2 git checkout .\Git-Befehle.html git push -u origin main // Inhalt neu auf Git laden ==> Befehl aus Github beim Erstellen vom Repositroy

Datei .gitignore

.gitignore ==> Datei für das Excluden von dateien, die nicht nach github übertragen werden.
